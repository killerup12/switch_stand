ЕСЛИ ИНТЕРНЕТ В WIFI ПЕРЕСТАЛ РАБОТАТЬ
================================================================

ВАЖНО: Сам Wi-Fi (подключение к роутеру) скорее всего работает —
страдает только выход в интернет. Поэтому ты можешь зайти в
веб-интерфейс роутера через браузер.

────────────────────────────────────────────────────────────────
ВАРИАНТ 1 — Полный откат через бэкап (САМЫЙ НАДЁЖНЫЙ)
────────────────────────────────────────────────────────────────

1. Открой браузер, зайди:
       http://192.168.88.1
   Логин: admin
   Пароль: (тот, что использовал при настройке)

2. Слева меню → Files

3. Найди файл:
       before-mihomo-20260425.backup

4. Кликни на него → справа кнопка "Restore"
   → подтверди → роутер перезагрузится (~1-2 минуты)

5. После перезагрузки всё вернётся к тому состоянию,
   которое было ДО изменений mihomo.
   PiHole, b4 продолжат работать как раньше.

────────────────────────────────────────────────────────────────
ВАРИАНТ 2 — Отключить только mihomo (если хочешь сохранить
            возможность позже включить)
────────────────────────────────────────────────────────────────

1. http://192.168.88.1 (admin / пароль)

2. Меню Containers
   → найди контейнер с именем "mihomo:latest"
   → выдели и нажми кнопку Stop (квадратик)

3. Меню IP → Firewall → вкладка Mangle
   → найди строки с комментарием, начинающимся на "mihomo-vpn"
   → выдели каждую → нажми "Disable" (D в тулбаре)
   ВАЖНО: НЕ удаляй, только Disable — потом можно вернуть

4. Меню IP → Routes
   → найди строки с комментарием "mihomo-vpn"
   → Disable

После этого весь трафик пойдёт через b4 как раньше.

────────────────────────────────────────────────────────────────
ВАРИАНТ 3 — Только сайты из VPN-списка не открываются,
            остальное работает
────────────────────────────────────────────────────────────────

Это НОРМАЛЬНОЕ поведение fail-close: если mihomo упал, эти
домены не открываются совсем (а не сливаются провайдеру).

Проверь, жив ли контейнер:
  Containers → mihomo:latest → должен быть со статусом "running"
Если нет:
  Выдели → Start (треугольник)

────────────────────────────────────────────────────────────────
ВАРИАНТ 4 — НИЧЕГО НЕ ОТКРЫВАЕТСЯ, даже WebFig
            (крайне маловероятно)
────────────────────────────────────────────────────────────────

1. Подключи Mac к роутеру кабелем (Ethernet) в любой порт
   КРОМЕ ether1 (ether1 — это кабель от провайдера).
   Используй ether2, ether3, ether4 или ether5.

2. http://192.168.88.1 — попробуй ещё раз

3. Если и это не работает — Hardware reset:
   • Зажми кнопку Reset на роутере на ~5 секунд
   • Роутер перезагрузится в дефолтную конфигурацию
   • Это СБРОСИТ ВСЁ — придётся настраивать заново
   • НЕ ДЕЛАЙ ЭТОГО, пока не попробовал всё остальное

БЭКАПЫ
================================================================

На случай, если файл бэкапа на роутере пропадёт, его копия
лежит на Mac:
       ~/Documents/mikrotik-backups/before-mihomo-20260425-212909.rsc

Это текстовый файл, его можно открыть и прочитать. Восстановить
можно так:
  1. Загрузить через Files в WebFig
  2. Открыть в Terminal/SSH:  /import file=имя_файла.rsc

Или через SSH:
   scp ~/Documents/mikrotik-backups/before-mihomo-20260425-212909.rsc admin@192.168.88.1:
   ssh admin@192.168.88.1 "/import file=before-mihomo-20260425-212909.rsc"
